# Drivee — система контроля аренды

Учёт коммерческой недвижимости: объекты, арендаторы, ежемесячная аренда,
автоматические напоминания об оплате и уведомления владельцу.

Заменяет ручную таблицу «Форма контроль аренды»: по каждому договору система
сама формирует ежемесячные начисления, за 3 дня до срока напоминает арендатору,
а при оплате/просрочке уведомляет владельца.

## Возможности
- **Объекты**: список, создание, редактирование, статусы (свободен / выложен в YKT / занят).
- **Аренда**: завести вручную или загрузить `.txt`. Параметры: арендатор, email,
  срок, **сумма аренды в месяц**, **расчётный день месяца**, № и дата договора.
- **Ежемесячные начисления**: создаются автоматически на весь срок договора.
- **Напоминание за 3 дня** до дня оплаты — письмо арендатору со ссылкой подтверждения.
- **Подтверждение оплаты** арендатором по ссылке → уведомление владельцу.
- **Учёт фактических оплат**, в том числе частичных (долги): начислено / оплачено / остаток.
- **Просрочка**: если к сроку не оплачено — письмо владельцу, статус «Просрочено».
- **Расторжение договора**: прекращает создание новых начислений.
- **Тестовые ручки писем** в карточке объекта — проверить SMTP без ожидания планировщика.

## Технологии
- Backend: Python, FastAPI, PostgreSQL, SQLAlchemy, APScheduler
- Frontend: React, Vite, Tailwind
- DevOps: Docker, docker-compose

## Быстрый старт (Docker)
```bash
cp .env.example .env      # заполнить SMTP_USER / SMTP_PASS
docker compose up --build
```
- Фронтенд: http://localhost:5173
- API + Swagger: http://localhost:8000/docs

При первом запуске на пустой БД автоматически создаются демо-объекты заказчика
из листа «2026» (Кирова 1/2/3, Пояркова 6п/7п, Б.М., Грозовая, Орджоникидзе 44).
Отключить: `SEED_ON_START=false`.

Чтобы наполнить БД историей за 2024–2026 (договоры арендаторов и помесячные
начисления с оплатами/долгами по форме контроля аренды) — поставьте
`SEED_HISTORY=true` (применяется к пустой БД) либо разово выполните
`docker compose exec backend python -m app.seed_history`. Email арендаторов в
истории демонстрационные (`@example.com`), уведомления по ним подавлены — реальные
письма не уходят.

## Настройка SMTP (письма)
В `.env`:
```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=you@gmail.com
SMTP_PASS=app_password        # для Gmail — пароль приложения, не основной
SMTP_USE_SSL=false            # при таймаутах на 587: SMTP_PORT=465 + SMTP_USE_SSL=true
SMTP_FORCE_IPV4=true
```
> `.env` содержит секреты и не коммитится (см. `.gitignore`). Перед передачей
> проекта смените пароль приложения SMTP на свежий.

## Как пользоваться (сценарий заказчика)
1. Открыть http://localhost:5173 — видны объекты.
2. В карточке объекта «Занять объект»: ввести арендатора, сумму, день оплаты, срок
   договора (или загрузить `.txt`). Появятся помесячные начисления.
3. Отмечать поступления кнопкой «Внести оплату» (поддерживаются частичные).
4. Система сама за 3 дня до срока напомнит арендатору, а при оплате/просрочке —
   уведомит владельца. Демонстрация без ожидания: `POST /api/jobs/run-now`.

## Локальная разработка
Backend:
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
# нужен запущенный postgres (docker compose up db) либо свой DATABASE_URL
uvicorn app.main:app --reload
```
Frontend:
```bash
cd frontend
npm install
npm run dev
```

## Тесты

## Основные API
| Метод | Путь | Назначение |
|-------|------|------------|
| GET/POST | `/api/properties` | список / создать объект |
| GET/PATCH/DELETE | `/api/properties/{id}` | карточка объекта |
| POST | `/api/properties/{id}/leases` | завести аренду (вручную) |
| POST | `/api/properties/{id}/leases/upload` | завести аренду из `.txt` |
| GET | `/api/properties/{id}/leases/{lid}/periods` | начисления по аренде |
| POST | `/api/properties/{id}/leases/{lid}/periods/{pid}/payments` | внести оплату |
| GET/POST | `/api/confirm/{token}` | подтверждение оплаты арендатором |
| POST | `/api/email/test/{tenant-reminder\|owner-paid\|owner-not-paid}` | тест писем |
| POST | `/api/jobs/run-now` | ручной прогон проверок |
| GET | `/api/health` | health-check |
